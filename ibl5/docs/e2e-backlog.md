# E2E Test-Quality Backlog

Catalogue of test-quality reduction opportunities in the Playwright suite (`ibl5/tests/e2e`) and the
Vitest api-e2e suite (`ibl5/tests/api-e2e`). Generated by a parallel multi-agent audit (2026-06-28) over
all 110 specs + 3 api-e2e files (~18.5K LOC) + helpers/fixtures/config; cross-file dedup and every final
"useless / doesn't-prove-functionality" call made by hand (the top-3 D items verified against source).

**Companion to** [`maintenance-backlog.md`](maintenance-backlog.md) and [`a11y-backlog.md`](a11y-backlog.md);
same emoji-coded status taxonomy.

**Format** (detailed entries):

```
### <id> <short title>
**Location:** path:line
**Problem:** what's wrong
**Suggested direction:** sketch, not a plan
**Est. effort:** S / M / L
**Risk if untouched:** what bug/cost this enables
**Status:** glyph + note
```

Effort scale:
- **S** тАФ single PR, < 1 day, one or few specs, no cross-cutting helper change
- **M** тАФ multi-spec / new shared helper / parametrization sweep, 1тАУ3 days
- **L** тАФ suite-wide pattern shift, > 3 days

## Status taxonomy

**Status** тАФ where the fix stands:
- тЬЕ **Implemented** тАФ resolved on `master` (verified on disk).
- тЧС **Partial** тАФ partially addressed; residual named in the note.
- ЁЯУЛ **Planned** тАФ a plan exists / PR open, not yet merged.
- тмЬ **Open** тАФ no plan yet.
- ЁЯЪл **Declined** тАФ deliberately won't-do (rationale in the note).

**Automouse-readiness** (for items not тЬЕ/ЁЯЪл):
- ЁЯЯй **Auto-mergeable** тАФ test-only, behavior-preserving for production (green-green); arms auto-merge unattended.
- ЁЯЯж **Automouse-safe, human-merge** тАФ implementable unattended but a gate-14 trigger forces `auto_merge: false` (none here yet тАФ test-only suite).
- ЁЯЯи **Conditional** тАФ would be ЁЯЯй except it **may surface a real production defect** when the assertion is strengthened (the happy path must be confirmed to actually pass first; if it doesn't, a real bug exists тЖТ separate fix + human triage), OR it needs one upfront scope decision (shared helper shape).
- ЁЯЯе **Not automouse-safe** тАФ irreducible judgment.

> **Why so many ЁЯЯи in Axis D:** strengthening a "doesn't-prove-functionality" test is green-green *only if the
> feature actually works*. Each D fix must first confirm the happy path passes the tightened assertion; a
> failure there is a genuine bug discovery, not a test regression. That branch needs a human.
