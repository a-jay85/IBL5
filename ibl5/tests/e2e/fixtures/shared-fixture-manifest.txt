# Shared-fixture drift manifest — curated seed rows the runtime fixture-drift
# gate snapshots before/after the chromium E2E run (bin/check-e2e-fixture-drift).
#
# Format: one snapshot query per non-comment line, pipe-delimited, EXACTLY 4 fields:
#
#     table | columns | order-by | where
#
# The bin script runs `SELECT <columns> FROM <table> WHERE <where> ORDER BY <order-by>`
# per line via `mariadb -N`, prefixes each result row with its table name, and diffs
# the before/after dumps. Any net difference is a durably-leaked shared-row mutation.
#
# CURATION RULES (load-bearing — a violation makes the gate blind or noisy):
#   - `columns` is an explicit business-value list — NEVER `*`, NEVER an
#     auto-increment PK, NEVER a wall-clock / NOW() column. Including a PK or a
#     timestamp makes the snapshot drift on EVERY run (false positive → gate gets
#     disabled). See bin/test-check-e2e-fixture-drift Case 5 (manifest hygiene).
#   - `order-by` must FULLY order the result set so identical snapshots are
#     byte-identical (determinism).
#
# CONSISTENCY CONTRACT: the tables/columns/baseline values below are coupled to
# tests/e2e/fixtures/ci-seed.sql (ibl_settings:45, ibl_fa_offers:462,
# ibl_trade_info:915/942, ibl_schedule NextSim rows:990) and to the reset-* actions
# in test-state.php (reset-extension:128, reset-fa-offers:454, reset-trade-offers:750,
# reset-fa-signings:782, reset-waiver-player:598, reset-rookie-option:623). If a
# future PR re-seeds any of these, re-derive the column list from BOTH files — do
# not guess. Enforces ADR-0052 (mutator isolation) at runtime.
#
